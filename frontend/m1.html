M1 — La Puerta de la Base (VS Code + Python) — GitHub-first

Historia

El Mentor Byte te da dos llaves: VS Code (tu cuaderno para escribir instrucciones) y Python (quien entiende y ejecuta esas instrucciones).

Hoy armamos el taller para toda la aventura. Si el taller queda bien, todo lo demás fluye.

¿Para qué sirve?

  • Abrir el proyecto en VS Code y confiarlo (trust).
  • Usar la terminal integrada en la carpeta exacta del proyecto.
  • Ver la versión de VS Code y de Python.
  • Subir evidencias a GitHub para revisión y nota.

Al final podrás…

  • Confiar la carpeta del proyecto.
  • Abrir la terminal en la ruta correcta.
  • Ver versiones de VS Code y Python.
  • Subir 3 evidencias a GitHub (en tu carpeta de estudiante) y verificar que se sincronizaron.

Material de aprendizaje sugerido

  • VS Code (introducción para principiantes)
  • Instalar VS Code en Windows 11 (paso a paso)

    • ¿Cómo trabajar con Git desde Visual Studio Code? — video en español (YouTube).

Práctica — Contrato

Paso 0: Fork → Clone → Rama m1-<usuario> → Commit/Push/Sync → Pull Request.

En tu proyecto local, crea la estructura:

students/<usuario>/
├─ docs/
├─ reports/
├─ data/bronze/
├─ data/silver/
├─ data/gold/
└─ tasks/

Agrega un .gitignore para no subir .venv, __pycache__, etc.

Define los entregables según tu perfil:

  • Ventas: docs/m1_vscode_ok.txt, docs/m1_workspace_ok.txt (con la línea exacta workspace: C:\\MinecraftShop\\PortalKidsVentas), docs/m1_notes.md.
  • Operaciones: docs/m1_vscode_ok.txt, docs/m1_workspace_ok.txt (línea exacta workspace: C:\\MinecraftShop\\PortalKidsOperaciones), docs/m1_notes.md.

Verifica que la terminal esté en la carpeta correcta, observa las versiones de VS Code y Python, haz commit, sincroniza y revisa que tus archivos estén en GitHub.

Entregables obligatorios

  • docs/m1_vscode_ok.txt
  • docs/m1_workspace_ok.txt
  • docs/m1_notes.md

Revisión en PR

  • Terminal en la ruta correcta.
  • Existencia de los 3 archivos con nombres exactos.
  • m1_workspace_ok.txt contiene la línea exacta.
  • Commit/Sync realizado.
