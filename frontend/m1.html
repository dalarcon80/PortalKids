M1 — La Puerta de la Base (VS Code + Python)

Historia

La Base Central se ilumina cuando llegan. En el centro, una mesa con dos piezas clave: una libreta brillante y un cristal con forma de serpiente.

“Bienvenidos”, dice el Mentor Byte. “Antes de abrir los cofres de datos, necesitamos un taller confiable. La libreta es VS Code, tu cuaderno inteligente para escribir instrucciones claras. El cristal es Python, el intérprete que entiende y ejecuta tu lenguaje. Sin estas dos piezas, el mapa de la tienda es solo ruido.”

Byte señala una compu: “Hoy aprenderán a confiar su carpeta del proyecto, abrirán la terminal en el lugar correcto y registrarán la versión de sus herramientas. Cuando el taller queda listo, todo lo demás es más fácil: los comandos funcionan, los archivos aparecen donde deben y ustedes avanzan sin perderse.”

¿Para qué sirve?

VS Code es el cuaderno inteligente para escribir programas. Python ejecuta tu código. Sin estas dos piezas no podrás leer ni transformar datos del caso real.

Al final podrás…

  • Abrir la carpeta del proyecto en VS Code y confiarla.
  • Abrir una terminal integrada en la ruta correcta.
  • Ver la versión de VS Code y de Python.
  • Dejar evidencias en docs.

Completa 5 fichas: editor, VS Code, intérprete, PATH, terminal. Para cada ficha incluye en bullets:

  • Definición (1–2 líneas, en tus palabras)
  • Ejemplo en este proyecto

  • Fuente vista (video)

Práctica — Contrato

Entrada: C:\\MinecraftShop\\minecraft_shop_project.

Logro: taller listo y evidencias.

Salidas:

  • docs/m1_vscode_ok.txt (escribe la versión de VS Code)
  • docs/m1_workspace_ok.txt con la línea exacta: workspace: C:\\MinecraftShop\\minecraft_shop_project
  • docs/m1_notes.md (tus 5 fichas)

Parte aplicada (detallada, sin código)

Situación: necesitarás ejecutar comandos del proyecto (activar el venv, correr scripts) siempre desde la carpeta correcta.

Qué debes hacer hoy:

  • Verifica que la terminal integrada abre en la ruta del proyecto; si no, muévela a esa carpeta y anota el comando que usaste.
  • Comprueba que VS Code está en PATH (puedes ver la versión por terminal o desde “Ayuda → Acerca de”).
  • Documenta en m1_notes.md cualquier ajuste que hiciste (1–2 líneas).

Meta de calidad: que mañana puedas crear el venv (M2) sin errores por rutas o PATH.

Entregables

m1_vscode_ok.txt, m1_workspace_ok.txt, m1_notes.md.

Verificación

Terminal en la ruta del proyecto; 3 archivos existen; la línea exacta está correcta.
